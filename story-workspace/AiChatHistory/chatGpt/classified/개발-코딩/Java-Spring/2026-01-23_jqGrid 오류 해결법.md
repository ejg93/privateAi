### 에러 핵심
```
Uncaught TypeError: Cannot read properties of null (reading 'id')
```

👉 **jqGrid가 row의 `id`를 읽으려고 했는데, 그 값이 null**이라서 터진 거야.

---

## 왜 이런 일이 생기냐면

jqGrid는 **각 행(row)에 반드시 고유한 id가 있어야 함**.  
그 id는 보통 다음 중 하나에서 가져와:

1. **JSON의 `id` 필드**
2. 또는 `colModel`에서 `key: true`로 지정한 컬럼

근데 지금은:

- SQL 결과가 **1행만 나오긴 하는데**
- **그 행의 모든 컬럼 값이 null**
- → jqGrid 입장에선 **row id = null**
- → 내부적으로 `row.id` 접근하다가 바로 폭발 💥

그래서 `jquery.jqGrid.min.js` 안에서 터지는 거고, 네 코드가 직접 잘못됐다기보단 **데이터 조건 문제**야.

---

## jqGrid가 기대하는 최소 조건

### ✔ 최소한 이것 중 하나는 반드시 있어야 함

#### 1️⃣ JSON에 id가 존재
```json
{
  "rows": [
    {
      "id": 1,
      "col1": null,
      "col2": null
    }
  ]
}
```

#### 2️⃣ colModel에 key 컬럼 지정
```js
colModel: [
  { name: 'SEQ', index: 'SEQ', key: true },
  { name: 'COL1' },
  { name: 'COL2' }
]
```

➡ 이 경우 `SEQ` 값이 **절대 null이면 안 됨**

---

## 네 상황에서 바로 쓸 수 있는 해결책들

### ✅ 해결책 1 (가장 정석)
**SQL에서 무조건 id용 컬럼 하나는 보장**

```sql
SELECT
    NVL(SEQ, 0) AS SEQ,
    COL1,
    COL2
FROM ...
```

또는 row 하나만 나오는 상황이면:
```sql
SELECT
    1 AS SEQ,
    NULL AS COL1,
    NULL AS COL2
FROM DUAL
```

---

### ✅ 해결책 2 (임시방편)
**jqGrid 로딩 전에 데이터 없으면 grid reload 안 하기**

```js
if (data.rows && data.rows.length > 0 && data.rows[0].id != null) {
    $("#grid").jqGrid('clearGridData')
              .jqGrid('setGridParam', { data: data.rows })
              .trigger('reloadGrid');
}
```

---

### ✅ 해결책 3 (서버에서 빈 row 자체를 안 보내기)
- SQL 결과가 **전부 null인 row면 아예 rows 배열을 비워서 응답**
```json
{
  "rows": []
}
```

jqGrid는 **빈 데이터는 잘 처리하지만**
👉 **id가 null인 row는 절대 못 참음**

## 지금 에러의 정체
```
Cannot read properties of null (reading 'SEQ')
```

👉 의미는 딱 이거야:

> jqGrid가 **row 객체에서 `SEQ` 값을 읽으려 했는데  
> row 자체가 null 이거나, SEQ가 null**이다

`key: true` 줬기 때문에 jqGrid는 **무조건 SEQ를 row id로 사용**하려고 해.

---

## 현재 구조를 정리하면

### 프론트
```js
colModel: [
  { name: 'SEQ', index: 'SEQ', key: true },
  { name: 'COL1' },
  { name: 'COL2' }
]
```

### 백엔드 / SQL
- 결과는 **1행**
- 근데 **모든 컬럼이 null**
- 즉:
```json
{
  "SEQ": null,
  "COL1": null,
  "COL2": null
}
```

👉 이 상태면 jqGrid는 **절대 로드 못 함**
(키 컬럼이 null이면 내부적으로 바로 터짐)

---

## 핵심 규칙 (jqGrid 불문율)
> **`key: true`로 지정된 컬럼은 null ❌ / undefined ❌**

row 하나만 있어도:
- 값이 없으면 ❌
- 0, 1, "DUMMY" 이런 값이라도 있으면 ⭕

---

## 바로 써먹을 해결책

### ✅ 해결책 1 — SQL에서 SEQ 강제 생성 (제일 추천)

#### row가 의미 없는 placeholder라면
```sql
SELECT
    1 AS SEQ,
    NULL AS COL1,
    NULL AS COL2
FROM DUAL
```

#### 실제 테이블이면
```sql
SELECT
    NVL(SEQ, ROW_NUMBER() OVER (ORDER BY 1)) AS SEQ,
    COL1,
    COL2
FROM ...
```

---

### ✅ 해결책 2 — JSON 단계에서 id 만들어주기
서버에서 jqGrid로 내려주기 전에:

```java
if (row.get("SEQ") == null) {
    row.put("SEQ", 1);
}
```

또는 loop 돌면서 index로라도 세팅.

---

### ✅ 해결책 3 — “빈 row” 자체를 보내지 않기
**이게 UI적으로 제일 깔끔한 경우도 많음**

```json
{
  "rows": []
}
```

jqGrid는:
- 데이터 없음 → 정상
- key 컬럼 null → 에러

## ✅ 제일 안전한 방식 (추천)
### `loadComplete`에서 SEQ 강제 세팅

```js
$("#grid").jqGrid({
  datatype: "json",
  colModel: [
    { name: 'SEQ', index: 'SEQ', key: true, width: 60 },
    { name: 'COL1' },
    { name: 'COL2' }
  ],
  loadComplete: function (data) {
    var grid = $(this);
    var ids = grid.getDataIDs();

    if (ids.length === 0) return;

    for (var i = 0; i < ids.length; i++) {
      var rowData = grid.getRowData(ids[i]);

      // SEQ가 없으면 강제로 생성
      if (!rowData.SEQ) {
        grid.setCell(ids[i], 'SEQ', i + 1);
        grid.setRowData(ids[i], { SEQ: i + 1 });
      }
    }
  }
});
```

✔ 화면 표시 + 내부 데이터 둘 다 안정  
✔ null row 터지는 거 방지

---

## ⚠️ formatter 방식 (보조용)
**화면에만 번호 보여주고 싶을 때**

```js
{
  name: 'SEQ',
  index: 'SEQ',
  key: true,
  formatter: function (cellValue, options, rowObject) {
    if (cellValue == null || cellValue === "") {
      return options.rowId || options.pos + 1;
    }
    return cellValue;
  }
}
```

❗ 하지만 이건 **id 생성 타이밍 이후**라  
👉 여전히 **초기 로딩 시 null이면 에러 날 수 있음**

---

## ✅ 진짜 깔끔한 방법 (UI용이면 이게 베스트)
**SEQ를 key로 안 쓰고, jqGrid 내부 id 사용**

```js
colModel: [
  { name: 'SEQ', index: 'SEQ', width: 60, formatter: 'rownum' },
  { name: 'COL1' },
  { name: 'COL2' }
]
```

이 경우:
- key 컬럼 없음
- jqGrid가 내부적으로 id 생성
- **null row여도 안 터짐**
