JavaScript에서 메모리 사용을 최소화하려면 즉시 실행되고 종료되는 함수를 만들 수 있습니다. 이를 즉시 실행 함수 표현식 (IIFE, Immediately Invoked Function Expression)이라고 합니다. IIFE는 함수를 정의하자마자 바로 실행되며, 그 실행 결과를 반환합니다.

IIFE를 만드는 방법은 다음과 같습니다:

```javascript
(function() {
  // 여기에 코드를 작성하세요.
  // 이 함수는 정의되자마자 실행되고 종료됩니다.
})();
```

이 형태의 함수는 자체 실행되므로 외부에서 참조할 수 없습니다. 따라서 이 함수 내에서 선언된 변수는 함수가 종료되면 메모리에서 해제됩니다. 이를 통해 메모리 사용을 줄일 수 있습니다.

예를 들어, 다음과 같이 IIFE를 사용하여 두 숫자의 합을 계산할 수 있습니다:

```javascript
(function() {
  var a = 10;
  var b = 20;
  var sum = a + b;
  console.log('합계:', sum);
})();
```

이 코드는 즉시 실행되며, a와 b 변수는 함수 실행이 종료되면 메모리에서 해제됩니다. 이렇게 IIFE를 사용하면 메모리 사용을 최소화할 수 있습니다.

호이스팅은 일반 함수 선언과 화살표 함수 간에 차이가 있습니다.

일반 함수 선언의 경우, 함수 선언을 전체가 호이스팅되므로 함수를 선언하기 전에도 호출할 수 있습니다.

}

};

    this.name = name;
  }

  speak() {
    console.log(`${this.name} makes a noise.`);
  }
}

class Dog extends Animal {
  constructor(name, breed) {
    super(name); // 상위 클래스의 생성자 호출
    this.breed = breed;
  }

  speak() {
    super.speak(); // 상위 클래스의 메서드 호출
    console.log(`${this.name} barks.`);
  }
}

// 출력:

3. `Yield`:
   `yield`는 제너레이터(generator) 함수 내에서 사용되며, 함수 실행을 중지하고 값을 반환하는데 사용됩니다. 제너레이터 함수는 중간에 멈추고 다시 시작할 수 있는 함수로, `yield` 키워드를 만나면 해당 값을 반환하고 함수의 실행을 멈춥니다. 이후 `next()` 메서드를 호출하여 제너레이터 함수를 다시 시작할 수 있습니다.

function* generatorFunction() {
  yield 1;
  yield 2;
  yield 3;
}

const generator = generatorFunction();
console.log(generator.next()); // 출력: { value: 1, done: false }
console.log(generator.next()); // 출력: { value: 2, done: false }
console.log(generator.next()); // 출력: { value: 3, done: false }
console.log(generator.next()); // 출력: { value: undefined, done: true }

위 예시에서 `yield` 키워드를 만나면 해당 값을 반환하고, 다음 `next()` 호출 시에 다음 `yield` 문으로 이동합니다. 마지막 `next()` 호출 시에는 `value`에 `undefined`가 반환되며, `done`은 `true`가 됩니다. 이로써 제너레이터 함수의 실행이 끝나고 종료됨을 알 수 있습니다.

   SELECT emp_id, emp_name

### ✅ 조건 분석:

\[
\sqrt{m^2 + 33} = n \quad (n \in \mathbb{N})
\]

양변 제곱:

\[
m^2 + 33 = n^2 \Rightarrow n^2 - m^2 = 33
\Rightarrow (n - m)(n + m) = 33
\]

이걸 만족하는 **자연수 쌍 (n - m, n + m)** 을 찾으면 된다.

---

### ✅ 33의 약수쌍:

33 = 1 × 33  
33 = 3 × 11  

따라서 가능한 경우는:

1. \(n - m = 1,\ n + m = 33 \Rightarrow\)
   \[
   2n = 34 \Rightarrow n = 17 \Rightarrow m = n - 1 = 16
   \]

2. \(n - m = 3,\ n + m = 11 \Rightarrow\)
   \[
   2n = 14 \Rightarrow n = 7 \Rightarrow m = n - 3 = 4
   \]

자연수 조건이므로 음수는 제외.

그런데 문제는 **정수 m 중 가장 작은 값**을 구하는 거니까, **음수인 경우도 고려**해야 한다.

---

### ✅ 음수도 고려:

음수 약수쌍도 존재:

- (-1, -33): \(n - m = -1,\ n + m = -33 \Rightarrow n = -17, m = -16\)
- (-3, -11): \(n - m = -3,\ n + m = -11 \Rightarrow n = -7, m = -4\)

→ 여기서 **가장 작은 m은 -16**

---

### ✅ 최종 계산:

가장 작은 \(m = -16\) 이고, 이 값을 \(a\)라 하면

\[
2a - 1 = 2 \times (-16) - 1 = -32 - 1 = \boxed{-33}
\]
