코드 자체는 “데이터(o 객체 값)를 버튼 HTML 문자열로 만들어서 화면에 뿌리는” 전형적인 JSP/JS 패턴인데, **핵심적으로 바뀐 부분은 `escHtml()` 적용**이라고 보면 된다.

---

## 🔍 코드가 하는 일 (전체 구조)
각 줄은 이런 식이야:

- `o.recUserName2` → 버튼 HTML 문자열 생성
- `o.recUserId2` → 버튼 HTML 문자열 생성
- 조건 만족 시 (`tabCode == "T05"` && `"미설치"`) → “변경” 버튼 생성

즉, **서버에서 받은 값을 버튼으로 렌더링 + 클릭 이벤트 연결**하는 코드.

---

## 🧠 바뀐 핵심 포인트

### 1. `escHtml()` 추가됨
```js
escHtml(o.recUserName2)
escHtml(o.recUserNm)
escHtml(o.recUserId)
```

👉 이게 제일 중요한 변경

### 왜?
**XSS (Cross-Site Scripting) 방지**

예를 들어 기존 코드가 이랬다고 가정하면:

```js
value='" + o.recUserName2 + "'
```

만약 값이 이런 식이면:

```text
홍길동' onclick='alert(1)
```

👉 결과 HTML:
```html
<input value='홍길동' onclick='alert(1)' ...>
```

→ **스크립트 삽입됨 (취약점)**

---

### 2. `escHtml()` 역할
`<`, `>`, `'`, `"` 같은 걸 escape 처리함

예:
```text
'  → &#39;
"  → &quot;
<  → &lt;
```

👉 즉,
**사용자 입력이 HTML/JS로 해석되지 않게 막음**

---

## 🧩 정리하면

### ✔ 고친 내용
- 모든 사용자/데이터 값에 `escHtml()` 적용

### ✔ 왜 고쳤냐
- **XSS 보안 취약점 막으려고**

### ✔ 효과
- 악의적인 문자열이 들어와도
- 버튼 HTML 깨지거나
- onclick에 코드 주입되는 것 방지

---

## ⚠️ 추가로 눈에 띄는 점 (중요)
이 코드 구조 자체가 사실 좀 위험한 방식이긴 함:

```js
onclick="openPop('값','값')"
```

👉 문자열 이어붙이기로 이벤트 만드는 구조라서  
**escape 빠지면 바로 취약점 터짐**

요즘은 보통 이렇게 안 하고:

```js
element.addEventListener(...)
```

이런 식으로 분리해서 처리함.
